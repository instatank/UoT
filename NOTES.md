## Mobile adaptation (portrait phones, ≤768px)

Decisions, each reasoned from the geometry's shape — not a shrink-to-fit:

- **Shell**: detail panel becomes a **bottom sheet** (rises on node selection, max 56dvh, tap the handle to tuck it, "detail ↑" pill to bring it back). Header wraps to two rows; the geometry switcher gets a full-width row of ≥44px buttons. Map fills the rest of the screen.
- **Radial — pan + auto-center, not shrink.** The full 800×800 map renders at 1:1 label scale inside a natively scrollable container (both axes); on entry it centers on the complaint, and each selection pans gently so the selected node sits in the band the sheet doesn't cover. Depth-as-radius stays literal and legible; the whole field is reachable by thumb pan. Chose native scroll over custom pinch-zoom: momentum and feel come free, zero gesture code.
- **River — stays horizontal; swipe along the river.** Considered rotating the metaphor vertical for portrait, rejected: Descent already owns the vertical axis, and the sandbox exists to compare geometries — two vertical flows would blur exactly the distinction being tested. Left→right is also load-bearing (source → reconvergence into the pool). The 1000×640 canvas renders near 1:1 and you swipe downstream. Flag for AA: this is my call, provisional until AA feels it (CLAUDE.md update only if confirmed as standing).
- **Descent — compact portrait strata.** Only geometry with a second layout: a 430-wide viewBox variant (same structure — shaft, alternating dig sides, rejected pocket, bedrock) that fits the viewport width exactly, so the only axis is down. Rows are taller (148 vs 80) and deepenings step *inward* toward the shaft (no room outward); rejected labels go below instead of right. ~2 screens of vertical travel to bedrock — scroll is the dig, no snap points, no feed mechanics.
- **Touch targets**: every MapNode gets an invisible hit circle (min r=26 SVG units ≈ ≥44px at rendered scales); the radial practice ring's transparent hit stroke widened 18→44. No functionality was hover-only (hover effects are cosmetic), so nothing needed a tap equivalent.
- **Safe areas**: `viewport-fit=cover` + `env(safe-area-inset-*)` on header, sheet bottom, arrival overlay, home.

Lessons:

- **Scroll room under the sheet**: with the sheet open, the map's natural scroll range ends with bottom content (practice at bedrock!) still hidden behind the sheet — caught because the Playwright tap on practice landed on the sheet. Fix: `margin-bottom: 58dvh` on the svg while the sheet is open, so any node can be panned above the sheet and tapped.
- The compact/desktop split in DescentGeometry is a `useCompact()` matchMedia hook (in `geometries/common.tsx`, breakpoint shared with SessionView's pan logic as `MOBILE_QUERY`). It initializes false to match the SSG'd desktop markup and corrects after hydration — a one-frame desktop-layout flash on phones is the accepted cost.
- Verified with Playwright (Chromium) at 390×844 and 360×780: full session loop to arrival in all three geometries, no horizontal page overflow, sheet/peek cycle, desktop 1440×900 regression. **Not verified on a real device**: iOS Safari dvh-vs-toolbar behavior, actual safe-area insets on a notched phone, momentum-scroll feel, backdrop-filter performance.

### The discoverability failure (AA's first real test) and the navigation pass

AA's field report: "the only thing that's happening is the complaint… I click, it opens, I close, it resets. THERE IS NOTHING ELSE AVAILABLE." Root cause: the sun sat ~75° below the beacon (physically outside the view), the guiding hint faded after 7 s, and the worlds were dim by design — so the universe visibly ended at the first stop. **The automated drive never caught it because it teleported between stops via the test hook; it verified the state machine, not the experience.** Standing lesson: for spatial UX, at least one test must complete the loop with real input only.

Fixes, all verified by a real-clicks-only Playwright run (zero hooks) plus a compass-only run and an idle run:

- **Layout arc**: the system now lies along a gentle forward descent — beacon ahead (camera starts on it), sun forward-below (~20°), worlds ringing the sun, gate directly *below* the sun. (First attempt put the gate *behind* the sun: occluded, and center-taps meant for it hit the sun. Same class of bug as the original — "next thing not visibly reachable.")
- **Gaze guidance**: after each release the camera's gaze eases onto the arc's next destination (re-aimed every frame — the lift-off drift otherwise pulls it off target). Any drag/wheel/arrow cancels it: guidance, not possession.
- **Cue halos** breathe on the next destination (same language as the maps' cue ring); arc-critical hints persist until acted on.
- **Pick scoring** normalizes by radius (d/rad) so a small gate under the cursor beats a huge sun covering the same pixels.
- **The compass** (bottom-right): hold chevrons to look, ◉ = "carry me onward" (travels to the cue, else nearest unvisited world). Arrow keys now *look* (W/space sails, S brakes). Pinch-dolly on touch; trackpad pinch (ctrl+wheel) sails.
- **Idle voyage**: ~7 s without input in free flight and the ship drifts (patiently, 0.35 rad/s max turn) toward the undiscovered; gravity completes the capture. Any input takes the helm back. Checked against decision 6: this is ambient drift toward *content*, not a reward loop; arrival remains the only exit.

## The Retreat (2026-07-13, branch `claude/wellness-retreat-ui-design-oeg8mp`)

AA's ask: "a realistic wellness nature retreat experience… immersive, interactive, intuitive… photo-realistic, 3D… voice, image, dynamic video, animation, and text." Built as the fifth presentation of the unchanged session state machine at `/retreat/[id]`: a hand-rolled WebGL valley (raw browser API — the no-new-dependency rule holds) where the complaint is a trailhead arch, the mechanism a spring, parallels are lineage sites (the rejected one a dry hollow with a guttering ember), and practice is a lake jetty; arrival is dusk + a rise over the water. Voice = the existing listen pills; sound = a generated outdoor bed (wind/birds/water/footfalls, `lib/retreat/nature.ts`); "dynamic video" = the living world itself (drifting clouds, moving water, swaying trees, day-phase light that leans toward dusk as the session deepens).

Lessons that cost real time:

- **The bog.** The fbm terrain dipped below the walkable waterline directly across the very first trail; the walker hit it, slid sideways forever (the axis-slide fallback "succeeds" with near-zero movement, so the give-up branch never fired), and the session froze. Fixes, both kept: trails are terrain-shaping (`heightAt` raises any tread crossing a wet dip into a causeway — the polyline is the single source of truth for the baked tread, tree exclusion, AND the walker), and the autopilot got a progress watchdog (a leg that stops shortening for ~2 s yields to free walk instead of shoving). Standing rule: *any procedural ground the player must cross needs a guaranteed-walkable spine, not a probabilistic one.*
- **dt clamps cause slow motion on slow devices.** The tick clamped `dt` at 60 ms; headless SwiftShader renders ~250 ms frames, so simulation time ran at 0.24× wall time and every walk crawled. Clamp generously (100 ms) — the clamp exists to survive hidden-tab catch-up, not to smooth slow devices.
- **Handedness bites once per engine.** A standard GL lookAt mirrors the Voyage's screen convention (yaw 0 = +z with east on screen-right is a *left-handed* view). The retreat builds its view-projection in closed form to match the Voyage exactly and disables face culling instead of fighting winding flips.
- **Spot-to-spot walks route along the trail network** (nearest trail → spring → target trail), so every guided journey literally passes through the mechanism — same invariant the radial map draws. Straight-line walks remain only for free ground taps.
- **Probe artifact worth remembering:** clicking a spot's projected anchor *during* the gaze-guidance turn misses (the camera rotates between projection and click). Probes must wait for the gaze to settle — a real user clicks what they currently see.
- `.v-sr-nav` needed `z-index` above the HUD: synthetic mouse clicks on the visually-hidden buttons were intercepted (keyboard/AT activation was never affected; the probe now drives it with Enter, which is what AT actually does).
- Verified: full loop by hook, by real clicks only, by compass alone (mobile touch), by keyboard under reduced motion; all three sessions; zero console errors. SwiftShader runs at the lowest adaptive tier (DPR 0.72, grass/motes off) — treat as worst-case old phone; the real-GPU pass is queued with AA's review.

### The character pass (same day) — AA's five refinements

AA on first look: "beautifully done… just as good, if not better, than I was hoping," then five asks: per-stop character, more elements, text presentation with character, easier navigation/discovery, human voices. Built (details in PLAN.md "The character pass"); lessons:

- **rAF's first timestamp can predate `performance.now()` taken in `start()`** — a negative `t` made a JS remainder negative, indexed a trail array at [-1], and the thrown exception killed the whole rAF loop (the re-arm lives after `tick()`). Cost an hour of misdiagnosis because it was timing-dependent and first-frame-only. Fixes: clamp `t ≥ 0` at the source, positive-modulo in indexers. Lesson pair: *any `x % 1` used as an array index needs a sign guard*, and *a per-frame loop should be assumed dead after one uncaught throw — look there first when "it renders one frame then freezes."*
- **`display: inline-block` collapses whitespace-only spans.** The carved/word-staged text dropped every space ("Somethingsarewithinourpower"). Whitespace units render as plain inline spans, animated units keep the block display. Also: per-character inline-blocks line-break mid-word, and phrase-length inline-blocks wrap as awkward boxes — carved became opacity-steps per char, turning became word-level.
- **Vertical faces need real ambient.** Lambert + `amb * (0.5 + 0.5 * N.y)` renders shadow-side walls as near-black voids in open daylight. Hemispheric ambient (0.68 floor) + a soft sky-bounce fill opposite the sun made the fortress/chapel read as stone. Terrain never showed it (mostly upward normals).
- **The stream that wanted to be a waterfall.** A rim-to-lake brook (carved channel + chained water pools) read as floating blue blobs on a knoll — the lake basin reached further west than assumed, and circular pool quads sit badly on slopes. Replaced with a **headland waterfall** dropping straight into the lake: one mound in `rawHeight`, sprite streaks/mist/foam, proximity roar. Simpler, robust from every angle, and visible clear across the lake from the jetty. Lesson: when scenery fights the terrain, move the scenery to where the terrain already wants it.
- Voice work is global: better voice selection + sentence chunking landed in `lib/voice.ts`, so the maps and Voyage read better too. Per-lineage registers are data (`lineageSpeakStyle`); the no-personas rule (decision 7) is annotated inline where future edits would see it.
- Re-verified after the pass: hook loop, real-clicks loop, compass-only mobile loop, reduced-motion keyboard loop, all three sessions, zero console errors; staged-text chambers screenshot-reviewed (typed/carved/illuminated/guttering).

## Retreat mobile pass (2026-07-13, after the mobile-polish + Retreat merge)

The Retreat was built and verified on desktop viewports; the mobile-polish pass was a sibling branch that never saw the Retreat (or the extra HUD button it added to the Voyage). A phone audit at 390×844/DPR 3 touch, 320px, and landscape caught four real seams, all fixed:

- **The HUD overflowed phones.** Four spelled-out pills ("← atlas · ♪ · ✦ voyage · ◈ bird's-eye") ran 419px wide on a 390px screen — bird's-eye sheared off the edge. Post-merge the Voyage had the same problem (the ⛰ retreat button pushed it to four controls; the mobile pass had verified three). Fix: labels wrapped in `span.vhud-label`, hidden ≤480px so the buttons collapse to 44px glyph circles (aria-labels carry the names). Desktop unchanged.
- **The hint pill could only ever be half the screen wide.** `.vhint` centered with `left: 50%; translateX(-50%)` — an abspos box shrink-fits against the space left of its offset, so the pill capped at 50vw and wrapped into a five-line lozenge on phones. Recentered with `left:0; right:0; margin-inline:auto; width:max-content` — now wraps only at its real 86vw limit (two lines on a phone). Lesson: *abspos `left:50%` centering silently halves the available width; inset-auto-margin centering doesn't.*
- **Touch targets followed width, not pointer.** A landscape phone (844×390) misses the ≤768px breakpoint and kept 36–38px HUD controls. The 44px floor (HUD pills, ♪, compass, listen pill) moved to `@media (pointer: coarse)` — orientation-proof. The ♪ toggle was 38px even in portrait; now 44px in the voyage/retreat HUD on any touch device.
- **The first hint said "arrow-keys" to thumbs** — same gap the Voyage's mobile pass fixed; the Retreat shipped from the sibling branch without it. Same `(pointer: coarse)` treatment.
- En route: compass chevrons now `setLookHeld` *before* `setPointerCapture` and capture in a try — capture throws `NotFoundError` on synthetic pointers (AT/automation) and the throw was killing the whole handler (both views). `⛰` carries U+FE0E everywhere so iOS renders a text glyph, not color emoji, in the monochrome HUD.
- Audit false alarm worth remembering: chamber `scrollWidth` > `clientWidth` flagged "horizontal overflow" — it's the `.chamber-mark` glyph bleeding at `right:-60px` behind `overflow-x: hidden`; clipped, not scrollable. Measure user-scrollability, not scrollWidth.
- Verified: full loop at 390×844 with the walk hook + real gestures only (tap the arch medallion on canvas → chamber; touch-tap stand up; chevron hold turns the camera; ◉ takes the helm); sweep at 320px, landscape 844×390, Voyage 390/320, desktop 1440×900 both views (labels intact, hint single-line bottom); session-header wrap with the retreat link at 390; zero console errors throughout. Still owed: AA's real-device pass (thermals, iOS Safari, walking feel) — unchanged.

## Corrections that mattered

- SVG node labels have `pointer-events: none` (so text never steals clicks) — any browser automation must click the `circle.core` inside `g.mnode`, not the label text. Also `wrapLabel` splits labels across separate `<text>` elements, so Playwright `hasText` across a wrapped phrase fails (textContent concatenates without spaces); match on a single word.
- MapNode label blocks: when the label sits *below* a node, the leading sublabel needs the block shifted down one line-height or it overlaps the circle. Radial deepening labels on horizontal spokes must go `above` their node — the parallel's label already owns that side.
- Label wrap is 18 chars × max 3 lines; longer node titles get silently truncated on the map (panel always shows the full title). Keep `title` fields ≤ ~45 chars.

## Environment (for future sessions in this container setup)

- Playwright is installed globally: run scripts with `NODE_PATH=/opt/node22/lib/node_modules`; Chromium is preinstalled (no `playwright install`).
- To kill a dev/prod server, use `fuser -k <port>/tcp` — `pkill -f next` matches the invoking shell's own command line and kills the session shell (exit 144).
- **A/B perf baseline recipe** (used for the atmosphere pass): `git worktree add <tmp> <baseline-sha>`, symlink the main checkout's `node_modules` into the worktree, `npm run build` there, `PORT=3001 npm run start`, then probe both ports with the same headless script sampling `window.__voyageEngine.frameEma`/`quality` after ~8 s and ~16 s of untouched flight (idle drift makes it deterministic-enough free flight). Headless Chromium rasterizes canvas on CPU (SwiftShader) — treat it as a worst-case old phone, not as representative of GPU devices.
- **Clone-time remote refs go stale.** This container's `origin/main` pointed at the repo's very first upload until an explicit `git fetch origin main` revealed the real head — fetch before comparing anything against `main`, or a baseline/merge decision starts from fiction.
- Fresh containers need `npm install` before `tsc`/`next build` (node_modules is not part of the clone).
